feat(sync): motor de sincronización continua, auto-reconexión P2P y merge por timestamp

Implementación de la fase final de estabilización del túnel P2P (PC <-> Móvil):

- Auto-Reconexión Inteligente: Implementado _reconnect_watcher y detección de caídas mediante fallos de heartbeat. El cliente móvil retiene el pairing en memoria y se reconecta silenciosamente.
- Demonio de Sync Global: Creado _global_vault_polling_daemon en main.py para asegurar que el móvil descargue y fusione contraseñas cada 15s siempre que haya conexión.
- Fusión (Merge) Precisa: Agregado upsert_from_bridge a la base de datos para respetar el updated_at original del PC. Esto corrige el bug de contraseñas no sincronizadas.
- Persistencia del Servidor: El host (PC) mantiene el servidor corriendo en segundo plano sin cerrarse al cambiar de pestaña en la interfaz.
- UI Reactiva: Integración de callbacks globales on_vault_sync que actualizan el Dashboard dinámicamente sin interrumpir la experiencia del usuario.
